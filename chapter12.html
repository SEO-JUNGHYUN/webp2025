Chapter 12 - 비동기 프로그래밍

비동기란?

  • 작업이 동시에 진행되며, 완료될 때까지 기다리지 않고 다음 작업을 실행함
  • 대표적인 예: 서버에서 데이터 받아오기, 파일 읽기 등

콜백 함수

  • 함수의 실행이 끝난 뒤 호출되는 함수
  • 중첩될 경우 "콜백 지옥"이 발생할 수 있음
function work(done) {
  setTimeout(() => {
    console.log("작업 완료");
    done();
  }, 1000);
}

work(() => {
  console.log("콜백 실행");
});

Promise

  • 비동기 작업을 처리하기 위한 객체
  • .then(), .catch()로 결과 처리
const promise = new Promise((resolve, reject) => {
  setTimeout(() => resolve("완료"), 1000);
});

promise
  .then(result => console.log(result))
  .catch(err => console.error(err));

async / await

  • 비동기 코드를 동기처럼 작성할 수 있게 해줌
  • await는 async 함수 안에서만 사용 가능
async function fetchData() {
  try {
    const res = await fetch("https://jsonplaceholder.typicode.com/posts/1");
    const data = await res.json();
    console.log(data);
  } catch (e) {
    console.error(e);
  }
}

fetchData();

정리

  • 비동기 처리를 위해 콜백, Promise, async/await 사용
  • 코드 가독성과 유지보수를 위해 async/await이 많이 쓰임
